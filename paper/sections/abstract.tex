\begin{abstract}\sloppy
Disaggregated prefill-decode (PD) serving improves LLM utilization by separating compute-intensive prefill from latency-sensitive decode, but introduces a new bottleneck: transferring the KV cache between stages over cross-node links. Existing systems send raw BF16 KV tensors without compression, while generic lossless codecs provide negligible compression on floating-point data and lossy formats such as FP8 sacrifice exactness.
We present SplitZip, a lossless, GPU-native KV cache compression codec for disaggregated PD serving. SplitZip exploits a numerical regularity of BF16 KV caches: exponent values concentrate on only 15--16 unique values across model families, with the top~8 covering 88--95\% of all elements. Leveraging this structure, SplitZip packs common exponents into 4-bit codes with a tiny escape stream for rare values, achieving $1.333\times$ lossless compression at 252/1460~GB/s encode/decode on GPU via Triton kernels. Integrated into the Mooncake Transfer Engine with layer-pipelined transfer, SplitZip achieves $1.16$--$2.27\times$ transfer speedup and $1.31$--$1.33\times$ end-to-end pipeline speedup across all tested bandwidth tiers. Evaluation on 5~models from 3~architecture families with 140~prompts confirms bitwise exact reconstruction and zero output degradation. An optional 3-bit near-lossless variant reaches $1.455\times$ compression with no observed quality impact.
\end{abstract}
